## What the App Does

Kaizen is a personal finance management web application. It allows authenticated users to track income and expenses, manage category-tagged budgets, monitor spending through rich dashboard visualisations, and export transaction data. The name reflects a continuous-improvement philosophy applied to personal financial health.

Core user journeys:
1. **Sign in** via Google OAuth.
2. **Onboard** — set a starting balance and choose a funding source, then configure initial budgets (or skip budgets and defer).
3. **Log transactions** — income or expense, with category, payment method, amount, date, optional notes, optional receipt attachment, and optional recurring schedule with reminder.
4. **Track budgets** — per-category budget limits with burn-rate visualisation; manual or AI-suggested setup.
5. **Explore insights** — spending summary, category breakdown, spending trends, and balance trends for any date range and payment-method filter.
6. **Manage categories** — create, edit, merge, or delete custom categories alongside system (global) categories.
7. **Manage payment methods** — wallets/cards named and tracked separately.
8. **Account settings** — appearance (light/dark/system theme), active session management, reminder preferences, tour reset.
9. **Goals** — page scaffold exists; feature is in progress (placeholder UI).

The app is an offline-capable SPA: transactions are queued in IndexedDB via Dexie and synced to the server when connectivity is restored.

## Key Features (Observable from Source)

### Authentication
- Google OAuth 2.0 sign-in only (no username/password).
- Persistent sessions stored in PostgreSQL; token hash cached in Redis; delivered as HttpOnly `SameSite=Lax` cookie.
- Session list and per-session revocation available in account settings.
- Google token revocation on logout.

### Onboarding
- Two-step wizard: Balance Setup → Budget Setup (skippable).
- Steps defined in `onboardingStep.ts`: `BALANCE | BUDGET | COMPLETE`.
- Draft state persisted in `onboardingDraftStorage`.
- Backend: `OnboardingProgressController` tracks step completion; `UserAccountController.completeOnboarding` issues persistent session.

### Dashboard (Home)
- 4-row bento grid layout:
  - Row 1: Net Worth / balance hero header
  - Row 2 (12-col grid): Spending Graph (5) · Wallet Bento (4) · Action Center (3)
  - Row 3 (12-col grid): Timeline Activity (7) · Wealth Profile Card (5)
  - Row 4 (3-col): Budget Burn Rate · Top Categories · Financial Intelligence
  - Row 5 (full): Subscription Watchdog
- Command Palette component for quick navigation.
- Guided dashboard tour via Driver.js (tour completion tracked as user flag).

### Transactions
- Full CRUD: create, list, detail, delete (bulk delete supported).
- Filter by type, category, payment method, date range.
- Free-text search with `SearchHighlight`.
- Sort persistence across sessions.
- Offline-first: entries queued in IndexedDB (`KaizenLocalDatabase`) with `syncStatus: pending | synced | failed`; `SyncManager` replays on connectivity restore using idempotent `clientGeneratedId`.
- Recurring transactions with `frequencyUnit` + `frequencyMultiplier`; backend `ReminderDeliveryJob` for notifications.
- Receipt / attachment upload (`ReceiptPicker`, `AttachmentViewer`).
- CSV export: applies active filter/search/sort pipeline, exports Date, Type, Amount, Description, Category, Payment Method.
- Balance history page.
- Selection action bar for multi-select bulk operations.

### Budgets
- Per-category, per-period (`WEEKLY | MONTHLY | YEARLY` — `BudgetPeriod` enum).
- Optimistic locking on `Budget` entity (`@Version`).
- Burn-rate thresholds: safe (<70%), high (70–100%), over (>100%).
- Manual budget setup page with category picker; batch create endpoint.
- Smart budget suggestion help URL configurable via env (`VITE_BUDGET_SUGGESTION_HELP_URL`).

### Categories
- System categories (`isGlobal=true`, seeded by `CategoryDataInitializer`) and user-created categories.
- Per-category icon and hex color from a design system (`CategoryDesignSystem`).
- Merge categories: all transactions reassigned, source deleted.
- `null` category_id = canonical "Uncategorized" state.

### Insights
- Endpoints: `/api/insights/summary`, `/category-breakdown`, `/trends`, `/balance-trends`.
- All support date range (`start`, `end`) and optional `paymentMethodIds` filter.
- Granularity: `DAILY | WEEKLY | MONTHLY` (default `MONTHLY`).
- Frontend: `InsightsPage` with `PeriodSelector`, `CategoryBreakdown` (chart), `SpendingTrends` (chart), `SpendingSummary`.

### Payment Methods
- Named wallets/cards; user creates and manages their own set.
- Seed data pre-populates common methods on account creation.
- Summary page shows per-method spending breakdown.

### Account / Profile
- Appearance: light / dark / system theme (persisted in `ThemeProvider` with localStorage).
- Sessions page: list active persistent sessions, revoke individual sessions.
- Reminders toggle (enables/disables recurring transaction notifications globally).
- Tour reset (re-runs dashboard onboarding tour).

### Goals
- `GoalDetailPage` exists as a placeholder scaffold. Feature is not yet implemented.
